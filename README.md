## Overview

This project benchmarks the cryptographic transition from classical RSA-2048 to post-quantum alternatives, using Shor's algorithm as the threat model.

The first part implements Shor's factoring algorithm from scratch in Qiskit — building a custom QFT, an IQFT, and a full QPE-based period-finding circuit. The algorithm is simulated on small composite numbers (N = 15, 21, 35) and the resulting qubit counts and gate metrics are used to extrapolate the resources needed to break RSA-2048. The empirical scaling model projects ~6,144 logical qubits are required, with a factoring time of under 5 hours on superconducting quantum hardware.

The second part benchmarks the post-quantum defense on classical hardware. Using the Open Quantum Safe `liboqs` library, it implements and times three cryptographic systems: the legacy RSA-KEM (baseline), the FIPS 203 standardized ML-KEM-512 (Kyber), and a Hybrid KEM that runs both RSA-2048 and ML-KEM-512 in parallel. Each system is run over 1,000 timed iterations and compared by key generation speed, encapsulation/decapsulation latency, and key and ciphertext byte sizes. Results show ML-KEM-512 is ~2.4× faster than RSA at encapsulation, with a public key of only 800 bytes, and that the Hybrid KEM adds negligible overhead over RSA alone.

## Repository Structure

### `codes/`
Contains all source code for the project.
- `shors_circuit_implementation.ipynb` — Interactive notebook implementing Shor's algorithm step by step: custom QFT/IQFT verification, circuit assembly, simulation, continued fractions post-processing, and qubit/gate scaling extrapolation.
- `pqc_benchmarking.ipynb` — Interactive notebook that compiles `liboqs` from source and runs the full RSA-KEM vs. ML-KEM-512 vs. Hybrid KEM benchmark suite with plots.
- `shors_impl.py` — Standalone Python script for running Shor's simulation and complexity analysis from the terminal.
- `shors_pqc_unified.py` — Unified pipeline script that runs both the quantum simulation and PQC benchmarks end to end and saves all result plots.

### `Results/`
Contains all output plots generated by the scripts and notebooks.
- `plot_quantum_scaling.png` — Qubit scaling regression and projected runtime for Shor's algorithm on RSA-2048.
- `plot_pqc_timing.png` — Operational runtime comparison (key generation, encapsulation, decapsulation) across all three systems on a log scale.
- `plot_pqc_sizes.png` — Public key and ciphertext byte size comparison across all three systems.
- `plot_hybrid_overhead.png` — Hybrid KEM overhead analysis comparing RSA alone, ML-KEM alone, theoretical sum, and actual Hybrid runtime.

### `Baseline_Research_Paper/`
Contains the reference paper this project validates against — Kinyua (2025), *EJ-COMPUTE_146.pdf*.

### `docs/`
Contains the project presentation slides in PDF and PowerPoint format.

### Root
- `Bridge_to_PQC_M_Danish_Raza_TM_Saeed.pdf` — The formal academic report for this project.
- `License` — MIT License.

## Setup

```bash
# Install Python dependencies
pip install qiskit==2.4.1 qiskit-aer==0.17.2 matplotlib numpy pycryptodome jupyter

# Compile liboqs (Linux/Colab only)
sudo apt-get install -y cmake gcc g++ libssl-dev ninja-build
git clone --depth 1 https://github.com/open-quantum-safe/liboqs
cmake -S liboqs -B liboqs/build -DBUILD_SHARED_LIBS=ON -DCMAKE_INSTALL_PREFIX=/usr/local
cmake --build liboqs/build --parallel 4 && sudo cmake --install liboqs/build
pip install liboqs-python

# Run the unified pipeline
python codes/shors_pqc_unified.py
```
